# Session Reflection & Documentation Update

Reflect on this session and update CLAUDE.md files to help future sessions work more efficiently.

## Your Task

1. **Analyze this session** - Review what was accomplished:
   - What task(s) were completed?
   - What key decisions were made?
   - What did you have to discover/explore that slowed you down?
   - Were there any misunderstandings or wrong turns?
   - Are there any 'gotchas' where things are not respecting standard practices?

2. **Identify reusable knowledge** - Ask yourself:
   - What architectural insights would help future sessions?
   - What patterns or conventions did I learn?
   - What "source of truth" or data flow knowledge was critical?
   - What services/files are important entry points?
   - What helped me the most to finish the task?

3. **Update CLAUDE.md** - Add only **general, reusable knowledge**:
   - Keep it concise - bullet points and short descriptions
   - Focus on "what you wish you knew at the start"
   - Avoid session-specific details or implementation patterns
   - Don't duplicate what's already there

4. **Decide which CLAUDE.md to update**:
   - `/CLAUDE.md` - Project-wide knowledge (monorepo structure, cross-package concerns)
   - Subpackage CLAUDE.md files for package-specific knowledge

## Guidelines

**DO add:**

- Architectural principles ("X is the source of truth")
- Key service/file locations for common tasks
- Important conventions or patterns
- Data flow relationships between components
- Testing patterns

**DON'T add:**

- Code examples or implementation details
- Session-specific task descriptions
- Information that's obvious from file names/structure
- Duplicate information

## Output

After updating, summarize:

1. What knowledge was added
2. Which file(s) were updated
3. Why this will help future sessions
