# 1:1 Agenda

Create agendas for 1:1s that matter — not status updates that could be async.

## What Good Looks Like

- **Report-driven** — The report owns the agenda; manager responds
- **Surfaces real issues** — Blockers, frustrations, career concerns — not just project updates
- **Time-appropriate** — 30-minute 1:1 ≠ 60-minute performance review
- **Forward-looking** — What you need, not just what happened
- **Builds trust** — Topics that strengthen the relationship, not just transactions
- **Actionable** — Each topic leads to a decision, commitment, or clarity

## What Bad Looks Like

- Status update that could be a Slack message
- Laundry list of everything you did this week
- Only tactical items, no relationship or career topics
- Manager does all the talking
- Same agenda every week with no evolution
- Topics too big for the time available

## Process

1. **Assess the relationship context:**
   - New relationship → More context-sharing, expectation-setting
   - Established relationship → Can go deeper, skip basics
   - Tension present → Address it directly, don't bury it

2. **Categorize what's on your mind:**
   - **Blockers** — What's slowing you down that your manager can help with?
   - **Decisions** — What do you need input or approval on?
   - **Feedback** — What do you want to share or receive?
   - **Career** — Growth, learning, trajectory questions
   - **Relationship** — How you work together, communication patterns
   - **Context** — Things they should know (but keep brief)

3. **Prioritize ruthlessly:**
   - Pick 2-4 topics max for a 30-minute 1:1
   - Lead with the hardest conversation, not the easiest
   - Career/relationship topics at least monthly, not never

4. **Frame each topic:**
   - What you want from this topic (decision, input, just awareness)
   - Brief context (1-2 sentences max)
   - Your current thinking (if you have one)

5. **Add standing items:**
   - Quick wins/celebrations (optional, keep under 2 min)
   - Open question: "Anything I should know?" / "Anything you need from me?"

## Output Format

```
## 1:1 Agenda — [Date]
**With:** [Manager/Report name]
**Duration:** [Time]

### Topics

1. **[Topic title]** — [Decision / Input / Awareness]
   [1-2 sentence context + your current thinking]

2. **[Topic title]** — [Decision / Input / Awareness]
   [1-2 sentence context]

### Quick Hits (if any)
- [Brief item that needs 30 seconds]

### Standing
- Anything I should know that I'm missing?
- What do you need from me this week?

### Notes
[Space for capturing discussion]
```

## Example

**Context:** IC engineer meeting with engineering manager, established relationship, mid-quarter

**Before (bad agenda):**

> - Standup updates
> - Ticket progress
> - Sprint planning
> - Anything else

**After (good agenda):**

```
## 1:1 Agenda — Tuesday 3pm
**With:** Alex (EM)
**Duration:** 30 min

### Topics

1. **Scope creep on auth project** — Decision
   Requirements expanded twice since kickoff. I can either push back on new asks or slip the deadline. I'm leaning toward pushing back. Want your read on stakeholder dynamics.

2. **Feedback on technical design reviews** — Input
   I've led 3 design reviews now. Curious what's landing and what I should adjust. Specifically wondering if I'm giving too much or too little context upfront.

3. **Tech lead path** — Awareness
   Want to plant a flag that I'm interested in the tech lead track. Not urgent, but wanted you to have it on your radar for planning.

### Quick Hits
- Heads up: taking Friday off

### Standing
- Anything I should know?
- What do you need from me?

### Notes
[To capture]
```

## Topic Ideas by Category

When you're stuck, scan these:

**Blockers/Tactical:**

- Decisions stuck in limbo
- Resource constraints
- Cross-team dependencies
- Unclear priorities

**Feedback:**

- Request feedback on recent work
- Share observations about team dynamics
- Discuss working style fit

**Career/Growth:**

- Skills you want to develop
- Projects you want exposure to
- Timeline questions
- Promotion criteria clarity

**Relationship:**

- Communication preferences
- How to handle disagreements
- What they need from you
- What you need from them

## Quality Checks

Before the 1:1:

- [ ] Would you be embarrassed if the agenda was just status updates?
- [ ] Is there at least one topic you're slightly nervous to bring up?
- [ ] Do you know what you want from each topic (decision/input/awareness)?
- [ ] Could any of these be handled async instead?
- [ ] Have you included career/relationship topics recently (within last month)?
- [ ] Is the hardest topic near the top, not buried at the bottom?

## Adapts To

**Report → Manager 1:1:**

- Report drives agenda
- Focus on what you need from them
- Surface blockers and career questions

**Manager → Report 1:1:**

- Report still owns agenda (you prompt, don't dictate)
- Create space for hard topics to surface
- Listen more than talk

**New relationship:**

- More time on working style, expectations
- Explicit check-ins on communication patterns
- Build trust before depth

**Tense relationship:**

- Address tension directly (avoiding makes it worse)
- Smaller topics, more frequent check-ins
- Focus on specific behaviors, not generalizations

## Common Mistakes

**All tactics, no relationship** — The 1:1 becomes indistinguishable from project standups. Career and trust topics matter.

**Avoiding hard conversations** — If you're never slightly uncomfortable, you're not using the 1:1 well.

**Overloading the agenda** — 8 topics in 30 minutes = nothing gets depth. Pick 2-4.

**Status dumps** — If it could be written up and sent async, do that instead.

**Manager talks the whole time** — 1:1s are for the report. Manager should mostly listen and respond.
