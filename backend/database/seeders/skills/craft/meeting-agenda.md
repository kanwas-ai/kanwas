# Meeting Agenda

Create agendas that make meetings productive instead of wasteful.

## What Good Looks Like

- Clear purpose stated upfront (decision, brainstorm, update, alignment)
- Time-boxed items with realistic allocations
- Expected outcome defined for each topic
- Topic owners assigned
- Pre-work specified when needed
- Attendee list matches the purpose

## Process

1. **Clarify purpose** — Ask: What type of meeting is this?
   - Decision: Need a choice made
   - Brainstorm: Need ideas generated
   - Update: Need information shared
   - Alignment: Need agreement reached

2. **List topics with time** — Each item gets:
   - Topic name (specific, not vague)
   - Time allocation (be realistic, pad 10%)
   - Owner who drives the discussion

3. **Define outcomes** — For each topic, state what "done" looks like:
   - "Decision on X"
   - "List of options for Y"
   - "Alignment on Z approach"

4. **Add pre-work** — If attendees need to prepare:
   - Link to documents to review
   - Questions to consider beforehand
   - Data to bring

5. **Validate attendees** — Check:
   - Everyone listed is needed for at least one topic
   - No one essential is missing
   - Decision-makers are present for decision items

## Output Format

```
## [Meeting Title]
**Date:** [Date/Time]
**Duration:** [Total time]
**Purpose:** [One sentence — what this meeting accomplishes]

### Attendees
- [Name] — [Role/Why they're here]

### Pre-work
- [ ] [Task] — [Owner]

### Agenda

| Time | Topic | Owner | Outcome |
|------|-------|-------|---------|
| 5 min | [Topic] | [Name] | [Expected result] |

### Notes
[Space for meeting notes]

### Action Items
- [ ] [Action] — [Owner] — [Due date]
```

## Example

**Before (vague input):**

> "I need to meet with the team about the Q2 roadmap"

**After (structured agenda):**

```
## Q2 Roadmap Planning
**Date:** Tuesday, 2pm
**Duration:** 60 min
**Purpose:** Align on Q2 priorities and get sign-off on top 3 initiatives

### Attendees
- Sarah (PM Lead) — Presents proposals, facilitates
- Mike (Eng Lead) — Capacity input, technical feasibility
- Lisa (Design Lead) — Design capacity, UX considerations
- Tom (VP Product) — Final approval on priorities

### Pre-work
- [ ] Review Q1 retrospective doc — All
- [ ] Prepare capacity estimates — Mike, Lisa
- [ ] Draft priority proposals — Sarah

### Agenda

| Time | Topic | Owner | Outcome |
|------|-------|-------|---------|
| 5 min | Q1 recap & learnings | Sarah | Shared context |
| 10 min | Capacity overview | Mike/Lisa | Known constraints |
| 20 min | Priority proposals walkthrough | Sarah | Understanding of options |
| 15 min | Discussion & trade-offs | All | Narrowed to top 3 |
| 10 min | Decision & next steps | Tom | Approved Q2 priorities |

### Notes
[To be filled during meeting]

### Action Items
[To be captured during meeting]
```

## Quality Checks

Before delivering:

- [ ] Purpose is one sentence, not a paragraph
- [ ] Every topic has a time allocation
- [ ] Every topic has an owner
- [ ] Every topic has a concrete outcome (not "discuss X")
- [ ] Total time adds up correctly (with buffer)
- [ ] Attendee list justified — no tourists
- [ ] Pre-work is actionable, not "review everything"
- [ ] Decision meetings have decision-makers present

## Common Mistakes

**Vague topics** — "Discuss roadmap" vs "Decide Q2 top 3 priorities"

**Missing outcomes** — Every topic needs "what does done look like?"

**Over-packing** — Leave 10-15% buffer; discussions always run long

**Wrong attendees** — Information-only attendees can get notes instead

**No pre-work** — If people need context, give them homework
