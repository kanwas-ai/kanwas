# Pyramid Principle

Apply Barbara Minto's Pyramid Principle to transform draft communication into clear, executive-friendly structure.

## The Methodology

**Source:** Barbara Minto, _The Pyramid Principle_ (McKinsey)

1. **Answer First** — Main point in the first sentence
2. **Group Supporting Ideas** — Related arguments cluster together
3. **MECE Structure** — Mutually Exclusive, Collectively Exhaustive
4. **Logical Order** — Time, structure, or importance order
5. **Vertical Logic** — Down answers "why/how", up answers "so what"

## Process

1. **Identify the core point** — What does the reader need to know or do?
2. **Extract supporting arguments** — Pull out all reasons, evidence, context
3. **Group into MECE categories** — No overlaps, no gaps
4. **Order by importance** — Most critical support first
5. **Rewrite answer-first** — Lead with conclusion, then support
6. **Trim ruthlessly** — Remove anything that doesn't support the main point

## Example

### Before

```
Hi team,

I wanted to give you an update on the Q3 roadmap planning. We had a meeting
with engineering last week and they raised some concerns about capacity.
Marketing also mentioned they need more lead time for launches. I've been
thinking about this and looking at our dependencies. There's also the issue
of the API migration that's been delayed. After considering everything, I
think we should reduce the Q3 scope to 3 features instead of 5 and push the
remaining 2 to Q4.

Let me know your thoughts.
```

### After

```
Hi team,

I recommend reducing Q3 scope from 5 features to 3, pushing 2 to Q4.

Three factors drive this:

**Capacity constraints**
- Engineering flagged bandwidth concerns
- API migration delay consumes additional cycles

**Launch dependencies**
- Marketing needs more lead time than current timeline allows

**Risk reduction**
- Fewer features = higher quality on what ships
- Q4 timing better aligns with dependencies

Let me know if you have concerns by Friday.
```

## Quality Checks

- [ ] Main point appears in first sentence
- [ ] Supporting points are grouped, not a flat list
- [ ] No more than 3-5 items per grouping level
- [ ] Output is shorter than or equal to original length
- [ ] Each group has a clear label or theme
- [ ] Reader can stop after first paragraph and know the ask
